## MISSION_TARGET

**Arbitrary cardinal-direction gravity for the player → true wall-walking.**

The flagship "elevated for the next long run" item (TODO.md #3). Gravity is
already a `Vec2` field in the physics layer (`BaseGravity.dir`, `GravityZone.dir`,
`GravityCtx::dir_at`) and non-player actors resolve a vector — but the **player**
collapses it to a scalar `gravity_sign` (`±1` on `vel.y`) at
`player_tick.rs:204` (`gravity_field.dir.y.signum()`, discarding `dir.x`). So the
player can only flip up/down, never walk on walls/ceilings.

Subsumes Jon's-List bugs: **#33** (movement broken / jump shorter when gravity
inverts) and **#55** (one-way platforms under gravity flip).

### Architecture

Parameterized gravity-axis (cardinal `gravity_dir: Vec2`), generalizing each
movement operation explicitly (NOT a coordinate transform — explicit is more
testable and avoids the coordinate-boundary bug class). Cardinal only keeps the
AABB collision axis-aligned (a 90° rotation of an AABB is an AABB).

- `down = gravity_dir`, `up = -gravity_dir`, `move_axis = perp(gravity_dir)`.
- Gravity accel + jump + fall-cap along `gravity_dir`; horizontal input along
  `move_axis`; "ground" = the contact whose normal opposes `gravity_dir`.
- One collision sweep runs along the gravity axis (ground/landing/one-way), the
  other along the movement axis (walls/cling).

`gravity_sign` stays for the vertical-only **actor** controllers (enemies, NPCs,
kinematic bodies); only the **player** model goes vector.
