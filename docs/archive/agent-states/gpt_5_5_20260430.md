## 1. Core objective

Build **Ambition**: a code-first, AI-generated 2D Metroidvania / platformer focused on exceptionally good movement, deep optional mechanics, and long-term extensibility. The game should be fun first even with simple geometry, raw collision boxes, procedural/generated audio, particles, and no prerendered assets.

The player-character/story premise is that the player is an AI-like entity discovering purpose, abilities, mathematics, ethics, collaboration, funding tradeoffs, and possibly multiple mirrored/alternate worlds. The “engine” should not care about the specific storyline; it should provide reusable movement/world rules that different games/storylines can assemble.

### Movement / abilities

* Sandbox should have all major optional abilities enabled by default.
* Abilities need to be independently enabled/disabled.
* Some future abilities might be incompatible.
* Need automated tests for movement/ability interactions.
* Blink:

  * quick eight-direction blink;
  * long-hold precision blink;
  * bullet time while aiming;
  * bullet time should affect world simulation, not input control.
* Important correction: the blink/fall bug was not fundamentally velocity clamping; it was a **two-clock simulation** issue.

  * Control time: real/unscaled for input and aiming.
  * Simulation time: scaled for gravity, velocity, enemies, moving platforms, particles.

* Rebound/bounce pads:

  * should affect enemies/dummies, not just player.
* Moving platform:

  * added partly to make bullet-time speed visibly comparable.
  * player should collide with and ride it.

## 5. Lessons learned

* **Do not mix input/control time with simulation time.**

  * Bullet time should slow gravity/platforms/enemies/particles.
  * It should not make aim cursor or button-hold thresholds unusable.
* **Post-blink velocity clamping alone is insufficient** and was not the real fix for the blink-fall bug.
* **Room transition spawns must be derived/validated**, not magic coordinates.

  * Bugs appeared where entering a vertical room spawned out of bounds or inside floor geometry, then snapped/teleported.
* **Transition cooldown should be a guard, not a freeze.**

  * Long door cooldown felt like frozen input after room load.
* **Edge exits should live on edges.**

  * Interior invisible triggers felt wrong.
  * Visible wall holes improve readability.
* **Door zones need explicit activation.**

  * Especially while flying, single `Up` conflicts with upward movement.
* **Enemy/dummy physics should respect world fixtures.**

  * Enemies ignoring rebound pads made the world feel inconsistent.
* **Fly toggle is persistent player state, not room state.**

  * It should not reset when changing rooms.
* **The Scroll Lab/Tangent Space entrance was cramped.**

  * Nearby blocks/platforms/rebound pads should leave breathing room near entrances.
* **Generated audio is viable for the assetless/code-first direction**, but a real audio crate may become worth it later.
* **The custom `Vec2` is probably technical debt.**

  * `glam::Vec2` or `bevy_math::Vec2` is likely better.
  * Do this as a dedicated migration, not mixed with gameplay patches.
* **The “engine” concept should be reframed.**

  * Avoid building a Unity/Bevy replacement.
  * Build a reusable deterministic rules core and Bevy adapter.

* Macroquad was used initially, then superseded by Bevy.

* Edge loading zones were initially inside rooms; corrected to be actual edge exits with wall holes.

* Global enemies/dummies in every room are intentional test fixtures for now, but likely superseded by explicit room enemy spawn specs later.

## 8. Compressed working state

Ambition is a Rust/Bevy, code-first 2D Metroidvania/platformer prototype. The priority is excellent, deep, replayable movement using simple generated geometry/audio/effects before rich art assets. The game story centers on an AI-like player discovering purpose, mathematical abilities, ethical tradeoffs, and possible mirrored worlds, but the reusable core should be story-agnostic.

Current code structure is a workspace with a reusable core-like crate (`ambition_engine`) and a Bevy sandbox (`ambition_sandbox`). Keep deterministic gameplay rules in core; keep rendering/audio/input/camera/debug in Bevy. Consider renaming to `ambition_core` later and adding an `ambition_bevy` adapter.

Current sandbox has a central hub plus Scroll Lab/Tangent Space, Vertical Shaft, Square Arena, and Tiny Chamber. Movement includes jump/double-jump, attack, dash/double-dash, pogo, wall cling/climb, blink/precision blink, bullet time, double-tap fastfall, fly, moving platform, rebound pads, particles, generated SFX, and generated ambience. Dummies exist in rooms as test fixtures.

Critical invariants:

* Use **two clocks**:

  * real control time for input/aim/hold gestures;
  * scaled simulation time for gravity/platforms/enemies/particles.
* While flying, `Up` is movement, so doors require **double-tap up**.
* Edge exits are automatic and should be visible wall holes at room boundaries.
* Interior doors require explicit interaction.
* Room transitions should go from one endpoint to the corresponding endpoint, not arbitrary coordinates.
* Transition arrivals should be validated/clamped/repaired so player never spawns out of bounds or embedded.
* Fly toggle should persist across room loads.
* Rebound pads affect enemies too.
* Entrance areas need breathing room.
* Avoid reinventing wheels:

  * migrate custom `Vec2` to `glam::Vec2` in a dedicated pass;
  * use `serde` + `ron`/`toml` for data-driven specs;
  * consider `kira` for serious adaptive audio.
