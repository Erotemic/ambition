# Ambition Sandbox — Web (wasm32) build

First-pass browser build for `ambition_sandbox`. Targets
`wasm32-unknown-unknown` and boots the visible Bevy app inside a `<canvas>`
with a statically embedded LDtk map and keyboard input.

This is intentionally a small slice. **Not yet supported:** audio, dev
tools / egui inspector, file watcher / hot reload, mobile touch, physics
debris (avian2d), browser save persistence. See [Scope](#scope) below.

## Scope

The `web` feature composes:

| Subsystem        | Status (first pass)                               |
| ---------------- | ------------------------------------------------- |
| LDtk runtime     | ✅ via `static_map` (embedded `sandbox.ldtk`)     |
| Keyboard / input | ✅ via `leafwing-input-manager`                   |
| Rendering        | ✅ via Bevy `webgl2` + canvas-backed winit        |
| Audio            | ❌ (kira/fundsp left out of the wasm dep graph)   |
| Dev tools        | ❌ (egui inspector + Bevy file watcher excluded)  |
| Mobile touch     | ❌ (`virtual_joystick` excluded)                  |
| Physics debris   | ❌ (avian2d excluded)                             |
| Save / settings  | No-op on wasm32 — kept in memory for the session  |

Browser save persistence (IndexedDB / LocalStorage via `web-sys`) is a
follow-up; the systems that would touch disk are cfg-gated to no-ops on
`target_arch = "wasm32"` (see `src/save.rs` and
`src/settings/persistence.rs`).

The full desktop and Android personas (`desktop_dev`, `android`, etc.)
are unchanged.

## One-time setup

```sh
# Rust toolchain target.
rustup target add wasm32-unknown-unknown

# wasm-bindgen CLI for the JS bootstrap files (`pkg/`).
# Pin the version to match the `wasm-bindgen` crate version in Cargo.lock
# to avoid the "wasm-bindgen CLI version mismatch" runtime error.
cargo install wasm-bindgen-cli --locked
```

## Compile-check only (fast feedback)

```sh
# Type-check the wasm build without producing JS bindings.
cargo check -p ambition_sandbox \
    --target wasm32-unknown-unknown \
    --no-default-features --features web
```

This is the equivalent of the desktop
`cargo check -p ambition_sandbox` smoke test; run it after any change
that touches the sandbox crate to keep the web build honest.

## Build + serve

```sh
# 1. Compile the wasm artifact (release recommended — debug wasm is huge).
cargo build -p ambition_sandbox --lib \
    --target wasm32-unknown-unknown \
    --no-default-features --features web \
    --release

# 2. Generate the JS shim + bg.wasm pair next to web/index.html.
wasm-bindgen \
    target/wasm32-unknown-unknown/release/ambition_sandbox.wasm \
    --out-dir crates/ambition_sandbox/web/pkg \
    --target web --no-typescript

# 3. Serve the directory (any static file server works; this one ships
#    with Python and needs no install). Then open
#    http://localhost:8000/ in a browser.
python3 -m http.server -d crates/ambition_sandbox/web 8000
```

Bevy's `AssetServer` on wasm fetches assets over HTTP relative to the page,
so any file the app loads from `assets/ambition/...` would need to be
served at `/assets/ambition/...`. The first-pass `web` feature uses the
embedded `static_map` build for the LDtk world; tilesets / sprites
referenced by `bevy_ecs_ldtk` may 404 until a follow-up wires up an
`assets/` symlink under `crates/ambition_sandbox/web/`.

## Bypassing the desktop CLI path

`ambition_sandbox::app::run_visible` does CLI parsing, the no-display-server
probe (`DISPLAY` / `WAYLAND_DISPLAY`), and a headless fallback. None of
that makes sense in a browser, so it is `#[cfg(not(target_arch = "wasm32"))]`.

The web build enters through `ambition_sandbox::web_start` (in `src/lib.rs`),
a `#[wasm_bindgen(start)]` shim the browser fires on its own once the
wasm module finishes instantiating. That shim installs the panic-to-console
hook and calls `app::run_web`, which builds the same plugin trio the
desktop binary does (`SandboxSimulationPlugin` + `SandboxLdtkPlugin` +
`SandboxPresentationPlugin`) with a canvas-pinned `WindowPlugin`.

## Where things live

- `crates/ambition_sandbox/web/index.html` — page + JS bootstrap.
- `crates/ambition_sandbox/web/pkg/` — generated by `wasm-bindgen` (git-ignored once added).
- `crates/ambition_sandbox/src/lib.rs` — `web_start` `#[wasm_bindgen(start)]` entry.
- `crates/ambition_sandbox/src/app/cli.rs` — `run_web()` app builder.
- `crates/ambition_sandbox/Cargo.toml` — `web`, `visible_web`, `web_platform` features.
