# Generator Instructions

Use the component sheet as a design vocabulary for a procedural rig, not as a final full animation sheet.

Positive style:

> polished cute white cartoon robot, thick clean black outlines, cyan glowing digital eyes, purple antenna and accents, rounded toy-like proportions, crisp reusable game sprite component, consistent lighting, isolated on a single solid green-screen background

Negative style:

> blurry, smudged, painterly, distorted, warped text, fake labels, inconsistent proportions, messy outlines, low resolution, grungy, over-detailed mechanical parts, death pose for hit reaction

Rules:

- Do not generate labels, grid lines, frame numbers, or timing text into production assets.
- Use solid green-screen or true transparency for component sheets.
- Keep enough spacing between components for connected-component extraction.
- Keep hit as recoil/stagger only; do not collapse the robot during hit.
- Use teleport as an action: idle, cyan particles gather, vanish, reappear.
- Use deterministic tooling for final sprite-sheet layout and metadata.
