# Metadata Refinement Workflow

## Problem

AI-generated component sheets can look polished, but their bounding boxes are not guaranteed to be production-ready. Hand-written YAML boxes are usually close but not exact: they may cut off antialiasing, miss small separated parts, or include a sliver of a neighboring component.

## Solution

Use two metadata stages:

1. `metadata/robot_components.rough.yaml`
   - Authored by hand or generated by an assistant.
   - Contains approximate boxes, semantic names, pivots, anchors, tags, and rig vocabulary.
   - Boxes should be close enough to indicate which component is intended.

2. `metadata/robot_components.refined.yaml`
   - Generated by `tools/robot_asset_tool.py refine`.
   - Uses the rough boxes as search windows.
   - Detects actual foreground by removing pixels close to the estimated green-screen color.
   - Runs connected-component analysis.
   - Selects components whose centers or sufficient overlap fall inside the rough box.
   - Writes corrected `rect` values plus `rough_rect` and refinement diagnostics.

## Green-screen removal

The source image uses a bright green background. The tool estimates the background color from border pixels, then uses a distance ramp:

- Pixels close to the background color become transparent.
- Pixels far from the background remain opaque.
- Intermediate pixels receive partial alpha to preserve anti-aliased edges.
- A small despill pass reduces green halos on foreground pixels.

## Avoiding neighboring sprites

For normal single-part components, rough entries set `keep_multiple_components: false` where appropriate. This prevents the refiner from including a neighbor if a rough box grazes it. For intentionally multi-part effects such as `fx_hit_particles` and `fx_dash_streak`, `keep_multiple_components: true` allows multiple disconnected foreground components to be retained.

## Recommended authoring rules

- Keep components separated by background pixels.
- Make rough boxes generous, but avoid enclosing the center of any neighbor.
- Use per-sprite `refine` overrides for effects, particles, and other multi-part sprites.
- Rebuild the contact sheet after every rough metadata edit.
- Treat pivots and anchors as rigging hints; they are automatically shifted into the refined crop coordinate system.
